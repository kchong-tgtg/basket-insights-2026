Business Question

What % of NL basket orders contain 2 or more units of the same SKU?
Raised by Sam Kashani in May 2026 to inform the trade pack / case pack optimisation hypothesis: if customers are already buying multiples of the same product, TGTG could negotiate larger case packs from suppliers, improving GP and reducing warehouse pick complexity.

Data Source
Table
tgtg.product_mart.fct_order_item_parcel — Redshift Serverless
Period
October 2025 – May 2026 (May 2026 is partial / month-to-date as of query run date)
Market
Netherlands only — filtered on country_iso_code = 'NL'
Order type
Basket orders only — filtered on is_basket_order = TRUE
Exclusions
Cancellations and refunds are excluded using COALESCE(is_cancellation, FALSE) = FALSE and COALESCE(is_refund, FALSE) = FALSE. Note: these fields are NULL (not FALSE) on normal orders, hence the COALESCE wrapper.
Metric Definition

A "multi-buy order" is defined as any order where at least one SKU has a total ordered quantity ≥ 2 within that same order.

-- Step 1: Aggregate quantity per (order, SKU)
SELECT order_id, tgtg_sku, SUM(quantity) AS total_qty
FROM tgtg.product_mart.fct_order_item_parcel
WHERE country_iso_code = 'NL'
  AND is_basket_order = TRUE
  AND COALESCE(is_cancellation, FALSE) = FALSE
  AND COALESCE(is_refund, FALSE) = FALSE
  AND paid_date_local >= '2025-10-01'
GROUP BY order_id, tgtg_sku

-- Step 2: Flag orders where any SKU has qty ≥ 2
SELECT order_id,
  MAX(CASE WHEN total_qty >= 2 THEN 1 ELSE 0 END) AS is_multi_buy
FROM above
GROUP BY order_id

-- Step 3: Compute rate
multi_buy_rate = SUM(is_multi_buy) / COUNT(DISTINCT order_id)
SKU-Level Table — How It Was Built

The Full Period SKU table shows the top 40 SKUs by multi-buy volume across the whole period. The monthly SKU tables (accessible via the month filter) show the top 50 SKUs per month, capped using a ROW_NUMBER() OVER (PARTITION BY month_key ORDER BY multi_orders DESC) window function to keep result sizes manageable.

Avg Qty is the average number of units of that SKU per order (across all orders that included that SKU, not just multi-buy orders). A value of 2.0+ means on average customers ordering this SKU order two or more — a strong signal for trade pack suitability. The AVG 2+ badge in the table flags these SKUs.

Known Limitations
May 2026 partial
May 2026 data is month-to-date (approximately 60–65% of the month). The rate and volume figures for May should not be compared directly to full months.
SKU rotation
The SKU catalogue changes month-to-month. A SKU appearing in the Jan top 50 but not in Feb top 50 does not mean it disappeared — it may have lower relative volume. The month filter shows rank within that month's data, not a fixed panel of SKUs.
No superdeal flag
fct_order_item_parcel does not contain a superdeal flag. Superdeal periods (which may temporarily inflate multi-buy rates due to pricing mechanics) are not controlled for in this analysis. Cross-reference with Smriti's superdeal sheet if isolating baseline behaviour.
Qty vs. unique SKUs
The metric counts 2+ units of the same SKU in one order, not 2+ different SKUs. An order with 1× Alpro Oat + 1× Lipton Ice Tea does not count as a multi-buy order under this definition.
